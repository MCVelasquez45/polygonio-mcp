## Security Review

### Improved In This Release

- Server request logs no longer print raw bodies.
- HTTP request logs include redacted structured context.
- HTTP responses include `x-request-id`.
- Socket connect/disconnect logs are structured.
- Shared redaction handles keys matching API key, secret, token, password, authorization, credential, or cookie.
- Regression coverage verifies that sensitive request-boundary fields are not emitted.

### Live-Trading Blockers

- Production authentication and authorization are not enforced across every operational endpoint.
- CORS remains permissive for local/paper-trading development.
- Socket.IO origin policy remains permissive.
- No global rate limiting is configured for exposed state-changing endpoints.
- Security headers are not centrally enforced through a production middleware.
- Credential rotation must be confirmed for any historical secret exposure.

These blockers should be treated as release blockers for live-money deployment, not as blockers for the V1 paper-trading baseline.
